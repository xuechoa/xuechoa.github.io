.

VMware 提供了三种网络连接模式：

1.桥接模式

虚拟机直接连接外部物理网络，主机起到了网桥的作用，也就相当于中间牵线搭桥的作用。在这种模式下虚拟机可以直接访问外部网络，并且对外部网络是可见的，也就是说以一种跟主机并列的级别出现；

在这种模式下，pc的ip和虚拟机的ip必须在同一个网段下，也就是ip的前三位必须一样。

这种模式的缺点是要占用ip但是路由器下分配的ip是有限的。

2.NAT模式(network address transition,网络地址转换)

虚拟机和主机构建一个专用网络，并且通过虚拟网络地址转换（NAT）设备对IP进行转换。虚拟机通过共享主机ip可以访问 外部网络，但是外部网络无法访问虚拟机。

internet通过路由器给我们的电脑分配ip，使其能连接网络并上网。nat模式下虚拟机的联网方式与此类似，相当于我们的电脑变成了一个外网，nat设备就相当于路由器，由该设备给虚拟机分配ip，所有虚拟机可以连接到我们的主机；

但是这种方式下主机是连接不到虚拟机的，就好像外网是直接连接不到我们的电脑的，怎么办呢？

我们的主机自己模拟出来一个网卡，这个网卡接到当前虚拟子网的路由器上，这样pc就和所有的虚拟机在同一个网段了，这个网卡就是VMnet 8.

这种模式下可以访问外网，但是外网不能访问我们，安全性比较高

3.仅主机模式

虚拟机只与主机共享一个专用网络，与外部网络无法通信。

与NAT模式相比仅仅是删去了那个路由器，不能访问外网了，但是依然虚拟了一个网卡，这个网卡就是VMnet 1.

.

linux 的诞生：

linus的类unix系统，所以叫linux。

Linux和Unix的渊源：

贝尔实验室

Ken Thompson—Unix之父

Dennis Ritchie

1969,ken主导开发出了一种多用户分时操作系统—Unix（对应之前开发失败的multics）

设计初衷：简洁高效

开发系统时觉得用汇编语言写太麻烦了，决定自己开发高级语言（dennis主导）： C语言。

unix 极其稳定，大型系统一般都使用unix系统，比如银行等大企业。

linux协议：如果你用了linux写内核，你写出来的东西也必须开源。

FreeBSD协议：如果你用了FreeBSD写内核，写出来的系统可以不开源，适用于商业化。

于是linus torvalds(李纳斯.托瓦兹)基于Minix从终端仿真器开始重新写了一个新系统—-linux，并开源。

—–git也是李纳斯写的，用于分布式代码管理。

.

GNU：GNU is Not Unix

1989 GPL协议： 一个自由软件发布的协议

只要遵循这个协议，那就说明该软件是完全开源的。

GPL维护用户的权力，而BSD更偏向公司和软件开发者的权力

深入理解计算机系统_计算机系统漫游

发表于 2022-05-26

第一节：计算机系统漫游

计算机系统由 硬件 和 系统软件 组成。

一个程序的生命周期如下：

第一步：源程序 或者源文件。这是程序员通过编辑器创建并保存的文本文件，后缀 .c 。

​ 系统中所有的信息：包括磁盘文件、内存中的程序、内存中存放的用户数据以及网络上传送的数据，都是由一串比特表示的。区别不同数据对象的唯一方法就是我们读到这些数据对象时的–上下文。

第二步：被编译器驱动程序翻译成一个可执行目标文件。该过程分四步完成：

​ 1.预处理器：将 .c 的源文件处理成.i 的源文件；

​ 2.编译器： 将处理后的.i 源文件 翻译成.s 的文本形式汇编程序；

​ 3.汇编器： 将.s 的文本形式的汇编程序处理成.o 的二进制的可重定位的目标程序；

​ 4.链接器： 将.o 的二进制的可重定位的目标程序链接为没有点后缀的二进制可执行目标程序。

经过链接的可执行目标文件可以被加载到内存中，由系统执行。

（二进制 -> 文本 -> 文本 -> 二进制）

（c ->i -> s -> o -> 空）

第三步： 此时之前的.c 源程序已经被编译系统翻译成了可执行目标文件并被存放在磁盘上。

（硬件知识）系统的硬件组成包括：总线，i/o设备，主存和处理器。

​ 1.系统主线负责在计算机的各个部件之间传输信息，包括数据，程序。

​ 2.i/o设备也就是计算机外设，包括显示器，鼠标，键盘，等等外接设备。

​ 3.主存是处理器执行程序处理数据时用来临时存储程序和数据的地方。其在物理上来说是由一组动态随机存储器（DRAM）芯片组成的。

​ 4. 处理器，也即中央处理单元（CPU）。这是解释存储在主存中的指令的引擎。处理器的核心是一个大小为一个字节的存储设备，称为程序计数器（PC）。在任何时刻，PC都指向主存中的某一条机器语言指令（即含有该指令的地址）。

操作系统–可以被看成是应用程序和硬件之间插入的一层软件，所有的应用程序对硬件的操作尝试都必须通过操作系统。

提问： 一个处理器如何能同时执行多个程序呢？比如看论文的同时听音乐，同时网盘后台还在下载视频。

回答：这是通过处理器在进程之间切换来实现的，这种机制叫做–上下文切换。

从一个进程到另一个进程的转换是由操作系统内核管理的，内核是操作系统主存中的常驻部分。

提问：请阐述进程和线程的区别与联系？

回答：通俗来讲，进程可以理解为完成一件事情的进度，比如跑够1km。

​ 而线程表示一件事情同时由几个人完成，比如上述的1km,如果是双线程，则可以安排一个人从起点出发，另一个人从500m处出发，两人同时起跑，合计跑够1km就行。多线程同理。但是体现在计算机上，线程只能处理器，几线程就意味着有几个处理器内核。

​ 一件事就有一个进程，几件事就有几个进程，但同一件事，也即同一个进程可以由多个线程完成。

文件就是字节序列，仅此而已。

​ 什么是操作系统？

​ Operating System:A body of software, in fact, that is responsible for making it easy to run programs(even allowing you to seemingly run many at the same time), allowing programs to share memory, enabling programs to interact with devices, and other fun stuff like that.(OSTEP)

概括一下：

1.能让程序跑起来。浏览器，编译器，IDE，游戏，外挂，任务管理器，杀毒软件，病毒，等等，都是程序；

2.甚至能让多个程序同时运行（具体原理后面会学到）；

3.程序共享；

4.能让程序连接外设，比如你的小U盘；

5.还有其他很多有趣的事。

更简单通俗来说，操作系统就是一段管理计算机的软硬件，为其程序提供服务的二进制代码。

程序视角： 对象+API（application programming interface,应用程序接口）

硬件视角： 一个C程序
