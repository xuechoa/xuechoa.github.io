关于博客使用的一些问题

发表于 2022-05-25
1.如何正确新建文档？

​ 首先，在博客所在根目录下的source/_posts 中直接新建文本文件或者粘贴文本文件是行不通的。

​ 正确的做法是：在博客所在根目录下右键选择 get Bush Here,在git的命令窗口输入以下命令：

hexo n “文件名” 。 其中的n代表new,表示新建一个文件。

​ 此时进入blog/source/_posts 文件夹，就可以发现你刚刚新建的文件。

2.如何将正确新建的文件同步到远端服务器？

​ 要将文件同步到远端服务器，必须先保证你要同步的文件是被正确建立的（方法参见问题1）。

​ 同步方法如下：

​ 第一步：在博客所在根目录下右键选择 get Bush Here,在git的命令窗口输入以下命令：

​ hexo g 。 其中的g代表generate,即生成的意思。这一步的作用是完成本地文件生成。

​ 第二步：在博客所在根目录下删去deploy_git 文件夹。

​ 第三步：在博客所在根目录下右键选择 get Bush Here,在git的命令窗口输入以下命令：

​ hexo d 。其中的d代表deploy,即部署的意思。这一步也解释了第二步的原因：因为要重新部署，所以要删去之前部署的内容，否则会出现代号108的错误。

​ 完成这三部曲之后，再登陆你的博客，就会发现你新建立的文件已经完成同步。

3.如何插入图片？

1.更改 _config.yml 配置。在博客所在根目录下右键选择 get Bush Here,在git的命令窗口输入以下命令：post_asset_folder: true ；

2.安装插件hexo-image-link. 在博客所在根目录下右键选择 get Bush Here,在git的命令窗口输入以下命令：npm install hexo-image-link –save ;

完成上述操作之后，每次新建文件都会随之自动新建一个同名文件夹。

插入图片时，先将要插入的图片放入该文件夹，然后再插入。

注意插入格式：./文件名（或者文件夹名，因为同名）/图片名.jpg（图片格式）
