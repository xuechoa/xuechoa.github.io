第一节：计算机系统漫游

计算机系统由 硬件 和 系统软件 组成。

一个程序的生命周期如下：

第一步：源程序 或者源文件。这是程序员通过编辑器创建并保存的文本文件，后缀 .c 。

​ 系统中所有的信息：包括磁盘文件、内存中的程序、内存中存放的用户数据以及网络上传送的数据，都是由一串比特表示的。区别不同数据对象的唯一方法就是我们读到这些数据对象时的–上下文。

第二步：被编译器驱动程序翻译成一个可执行目标文件。该过程分四步完成：

​ 1.预处理器：将 .c 的源文件处理成.i 的源文件；

​ 2.编译器： 将处理后的.i 源文件 翻译成.s 的文本形式汇编程序；

​ 3.汇编器： 将.s 的文本形式的汇编程序处理成.o 的二进制的可重定位的目标程序；

​ 4.链接器： 将.o 的二进制的可重定位的目标程序链接为没有点后缀的二进制可执行目标程序。

经过链接的可执行目标文件可以被加载到内存中，由系统执行。

（二进制 -> 文本 -> 文本 -> 二进制）

（c ->i -> s -> o -> 空）

第三步： 此时之前的.c 源程序已经被编译系统翻译成了可执行目标文件并被存放在磁盘上。

（硬件知识）系统的硬件组成包括：总线，i/o设备，主存和处理器。

​ 1.系统主线负责在计算机的各个部件之间传输信息，包括数据，程序。

​ 2.i/o设备也就是计算机外设，包括显示器，鼠标，键盘，等等外接设备。

​ 3.主存是处理器执行程序处理数据时用来临时存储程序和数据的地方。其在物理上来说是由一组动态随机存储器（DRAM）芯片组成的。

​ 4. 处理器，也即中央处理单元（CPU）。这是解释存储在主存中的指令的引擎。处理器的核心是一个大小为一个字节的存储设备，称为程序计数器（PC）。在任何时刻，PC都指向主存中的某一条机器语言指令（即含有该指令的地址）。

操作系统–可以被看成是应用程序和硬件之间插入的一层软件，所有的应用程序对硬件的操作尝试都必须通过操作系统。

提问： 一个处理器如何能同时执行多个程序呢？比如看论文的同时听音乐，同时网盘后台还在下载视频。

回答：这是通过处理器在进程之间切换来实现的，这种机制叫做–上下文切换。

从一个进程到另一个进程的转换是由操作系统内核管理的，内核是操作系统主存中的常驻部分。

提问：请阐述进程和线程的区别与联系？

回答：通俗来讲，进程可以理解为完成一件事情的进度，比如跑够1km。

​ 而线程表示一件事情同时由几个人完成，比如上述的1km,如果是双线程，则可以安排一个人从起点出发，另一个人从500m处出发，两人同时起跑，合计跑够1km就行。多线程同理。但是体现在计算机上，线程只能处理器，几线程就意味着有几个处理器内核。

​ 一件事就有一个进程，几件事就有几个进程，但同一件事，也即同一个进程可以由多个线程完成。

文件就是字节序列，仅此而已。
